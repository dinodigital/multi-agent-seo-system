# Мультиагентная система создания SEO-статей

Система из 8 AI-агентов для создания качественных SEO-статей, которые привлекают органический трафик и лиды.

## Как это работает

```
┌─────────────────────────────────────────────────────────────┐
│                      ОРКЕСТРАТОР                            │
│                       CLAUDE.md                             │
└─────────────────────────────────────────────────────────────┘

     ┌──────────────────────┼──────────────────────┐
     ▼                      ▼                      ▼
┌─────────┐          ┌─────────────┐         ┌──────────┐
│ Агент 1 │    →     │  Агент 2    │    →    │ Агент 3  │
│Семантика│          │ Конкуренты  │         │  Бриф    │
└─────────┘          └─────────────┘         └──────────┘
                                                   │
                                          ⏸️ КОНТРОЛЬНАЯ
                                             ТОЧКА
                                                   │
┌─────────┐          ┌─────────────┐         ┌──────────┐
│ Агент 8 │    ←     │  Агент 7    │    ←    │ Агент 6  │
│Формат   │          │  Редактор   │         │   SEO    │
```

## Быстрый старт

### 1. Клонируй репозиторий

```bash
git clone [url]
cd multi-agent-seo-system
```

### 2. Открой в Claude Code

```bash
claude
```

### 3. Настрой систему под свой продукт

```
> setup
```

Система задаст вопросы о твоём продукте и автоматически заполнит файл `knowledge/product_info.md`.

### 4. Создай первую статью

```
> create article: как увеличить продажи
```

## Команды

| Команда | Описание |
|---------|----------|
| `setup` | Настроить систему под свой продукт |
| `create article: [тема]` | Создать одну статью |
| `batch: [N]` | Создать N статей из semantic_core.csv |
| `show progress` | Показать статус работы |

## Агенты

| # | Агент | Задача |
|---|-------|--------|
| 1 | Семантический аналитик | Анализ запроса, интента, связи с продуктом |
| 2 | Исследователь конкурентов | Анализ топа выдачи, паттерны, незакрытые ниши |
| 3 | Создатель брифа | Детальный план статьи с тезисами и примерами |
| 4 | Копирайтер | Написание текста по брифу |
| 5 | Эксперт по продукту | Проверка упоминаний продукта, добавление конкретики |
| 6 | SEO-оптимизатор | Метатеги, ключевые слова, рекомендации по изображениям |
| 7 | Редактор | Финальная вычитка, исправление ошибок |
| 8 | Форматировщик | Создание готовых файлов article.md и meta.md |

## Подготовка к работе

### 1. Заполни информацию о продукте

Открой `knowledge/product_info.md` и заполни:

- Название и описание продукта
- Целевая аудитория
- Боли клиентов
- Функционал
- Преимущества
- Тарифы

Или используй команду `setup` — система поможет заполнить интерактивно.

### 2. Подготовь семантическое ядро

Отредактируй `input/semantic_core.csv`:

```csv
keyword,volume,difficulty,intent,priority
как увеличить продажи,12500,65,коммерческий,высокий
crm система для бизнеса,8200,58,коммерческий,высокий
...
```

Колонки:
- `keyword` — ключевой запрос (обязательно)
- `volume` — объём поиска (опционально)
- `difficulty` — сложность продвижения (опционально)
- `intent` — тип интента (опционально)
- `priority` — приоритет (опционально)

```

    ├── 01_semantic_analysis.md
```

### article.md
Готовый текст статьи в Markdown — копируй и публикуй.

### meta.md
- SEO-метаданные (Title, Description, URL)
- Рекомендации по изображениям с alt-текстами
- Внутренние и внешние ссылки
- Пошаговая инструкция по публикации

## Контрольная точка

После создания брифа (агент 03) система останавливается и ждёт одобрения:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚠️ КОНТРОЛЬНАЯ ТОЧКА

• Объём: 2500 слов
• Разделов: 6
• Ключевые темы: [...]

Файл: _workflow/03_brief.md

✅ `approve` — продолжить
✏️ [отредактируй файл и напиши "continue"]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Это позволяет:
- Проверить план перед написанием
- Внести правки в бриф
- Убедиться что направление верное

## Советы

### Качество статей
- Чем детальнее `knowledge/product_info.md`, тем лучше статьи
- Добавляй реальные кейсы и цифры
- Описывай функции конкретно, не абстрактно

### Семантическое ядро
- Начинай с высокочастотных коммерческих запросов
- Группируй близкие темы
- Учитывай сезонность

### Оптимизация процесса
- Проверяй бриф на контрольной точке
- Вноси правки до написания, а не после
- Используй `batch` для серийного производства
